# v0.1.1-alpha.1 (Draft)

Release type: post-alpha stabilization / hardening

This release focuses on reliability fixes after `v0.1.0-alpha.0`.

## Highlights

- approval outcome -> business status sync
- overnight attendance normalization starter hardening
- import/export asynchronous worker processing
- stronger smoke flow coverage for login + core screens + core APIs
- release/ops documentation hardening

## Stabilization Changes

### 1) Approval Outcome Synchronization

Approval terminal states now propagate to linked business records by `documentId`:

- `LeaveRequest` status sync
- `AttendanceCorrection` status sync

Status mapping:

- approval `APPROVED` -> business `APPROVED`
- approval `REJECTED` -> business `REJECTED`
- approval `CANCELED` -> business `CANCELED`
- approval `DRAFT` / `IN_REVIEW` -> business `REQUESTED`

Audit logs are written for each synchronized mutation.

### 2) Attendance Normalization Hardening

- overnight shift policy windows are now supported in starter flow
- post-midnight events are grouped to the correct prior work date for covered overnight scenarios
- raw attendance events remain immutable source records
- ledger generation continues to persist policy version and source-event linkage

### 3) Import/Export Workerization

Import/export execution is now asynchronous:

- API create endpoints enqueue jobs as `PENDING`
- worker claims and processes jobs (`RUNNING` -> `SUCCEEDED|FAILED`)
- import keeps row-level validation/error reporting (`ImportJobRow`)
- export persists result file linkage (`ExportJob.resultFileId`) and summary metadata

### 4) Smoke Coverage Baseline Upgrade

`scripts/smoke-stack.sh` now additionally verifies:

- auth login + `/auth/me`
- API visibility checks for documents/approvals/attendance ledger/expenses
- web route load checks for `/`, `/documents`, `/approvals`, `/attendance/ledger`, `/expenses`
