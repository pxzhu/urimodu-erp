# ERP Role Center 와이어프레임 v1

작성일: 2026-03-18
목적: 우리모두 ERP의 홈/결재/근태/경비 화면을 역할 중심 ERP UX로 바꾸기 위한 텍스트 와이어프레임

---

## 1. 관리자 홈 `/workspace`

```text
┌─────────────────────────────────────────────────────────────────────┐
│ [역할 배지] 관리자 업무 허브                                       │
│ 오늘 우선 처리: 승인 지연 / 근태 검토 / 경비 증빙 누락             │
│ [결재함 열기] [근태 검토] [경비 확인]                              │
├─────────────────────────────────────────────────────────────────────┤
│ KPI 1 직원수 │ KPI 2 결재 대기 │ KPI 3 근태 검토 │ KPI 4 경비 진행 │
├─────────────────────────────────────────────────────────────────────┤
│ 운영 경고                                                          │
│ - 지연 승인 4건                                                    │
│ - 근태 검토 필요 7건                                               │
│ - 증빙 누락 경비 3건                                               │
│ - 실패한 배치/연동 1건                                             │
├─────────────────────────────────────────────────────────────────────┤
│ 오늘 처리할 큐                                                     │
│ [긴급] 대표 결재 2건                                               │
│ [검토] 근태 정정 5건                                               │
│ [일반] 비용 승인 6건                                               │
├─────────────────────────────────────────────────────────────────────┤
│ 빠른 실행                                                          │
│ [직원 등록] [문서 생성] [결재선 설정] [가져오기] [증빙 업로드]     │
├─────────────────────────────────────────────────────────────────────┤
│ 최근 활동 / 병목 현황                                              │
│ 최근 승인, 반려, 문서 상신, 실패한 import/export 등                │
└─────────────────────────────────────────────────────────────────────┘
```

### 의도
- 단순 모듈 진입보다 “오늘의 운영 우선순위”를 보여주는 것이 핵심
- 관리자에게는 KPI보다 예외/지연/병목이 더 중요함

---

## 2. 일반 직원 홈 `/workspace`

```text
┌──────────────────────────────────────────────────────────────┐
│ [역할 배지] 개인 업무 홈                                     │
│ 오늘 할 일: 내 문서 / 내 결재 / 내 근태 / 내 휴가            │
│ [휴가 신청] [문서 작성] [근태 정정]                          │
├──────────────────────────────────────────────────────────────┤
│ KPI 내 결재 진행 │ KPI 내 휴가 잔여 │ KPI 내 경비 청구 │ KPI 알림 │
├──────────────────────────────────────────────────────────────┤
│ 내 작업 큐                                                     │
│ - 결재 보완 요청 1건                                           │
│ - 영수증 미첨부 경비 1건                                       │
│ - 근태 정정 필요 1건                                           │
├──────────────────────────────────────────────────────────────┤
│ 추천 액션                                                      │
│ [문서 상신 이어서 하기] [경비 제출 마무리] [휴가 일정 확인]     │
├──────────────────────────────────────────────────────────────┤
│ 최근 활동                                                      │
│ 내가 상신한 문서 / 승인 결과 / 알림 로그                       │
└──────────────────────────────────────────────────────────────┘
```

### 의도
- 직원에게는 운영 전체보다 내 업무 맥락이 중요함
- 셀프서비스형 ERP 느낌을 강화해야 함

---

## 3. 결재함 `/approvals`

```text
┌────────────────────────────────────────────────────────────────────┐
│ 결재함                                                            │
│ 설명: 대기 중인 결재를 우선순위 기준으로 빠르게 처리             │
├────────────────────────────────────────────────────────────────────┤
│ KPI: 대기 12 │ 긴급 3 │ 검토 필요 4 │ 오늘 처리 완료 5           │
├────────────────────────────────────────────────────────────────────┤
│ 필터바: [전체] [긴급] [검토필요] [일반] [승인] [반려]            │
├────────────────────────────────────────────────────────────────────┤
│ 긴급 큐                                                           │
│ ┌ 카드 ─ 문서명 / 현재 단계 / 상신 시각 / 담당자 / 승인·반려 ┐  │
│ └─────────────────────────────────────────────────────────────┘  │
├────────────────────────────────────────────────────────────────────┤
│ 검토 필요 큐                                                      │
│ ┌ 카드 ─ 반려 이력 / 코멘트 있음 / 단계 지연 / 액션 ──────────┐  │
│ └─────────────────────────────────────────────────────────────┘  │
├────────────────────────────────────────────────────────────────────┤
│ 일반 큐                                                           │
│ 테이블 또는 카드 리스트                                           │
└────────────────────────────────────────────────────────────────────┘
```

### 카드 상세 정보
- 문서명
- 문서 상태
- 결재선 현재 단계
- 현재 결재자
- 상신 후 경과 시간
- 다음 행동 버튼

### 의도
- “목록 확인”이 아니라 “바로 처리”에 초점
- 긴급/검토/일반을 나누면 사용자가 판단하기 쉬워짐

## 4. 근태 원장 `/attendance/ledger`

```text
┌────────────────────────────────────────────────────────────────────┐
│ 근태 원장                                                         │
│ 설명: 정규화된 근태 데이터 + 검토 필요 예외 우선 확인           │
├────────────────────────────────────────────────────────────────────┤
│ KPI: 총 근무일 │ 검토 필요 │ 지각/누락 │ 정책 미적용            │
├────────────────────────────────────────────────────────────────────┤
│ 예외 카드                                                         │
│ - 출근 누락                                                       │
│ - 퇴근 누락                                                       │
│ - 정책 버전 mismatch                                              │
│ - 비정상 장시간 근무                                              │
├────────────────────────────────────────────────────────────────────┤
│ 필터: [검토 필요만] [오늘] [이번주] [직원] [정책]                │
├────────────────────────────────────────────────────────────────────┤
│ 테이블                                                            │
│ 근무일 / 직원 / 상태 / 출퇴근 / 근무시간 / 정책 / 원본 이벤트수 │
└────────────────────────────────────────────────────────────────────┘
```

### 의도
- 원장은 원장대로 두되, 실무자는 예외만 빨리 보고 싶어함
- 기본 필터를 “검토 필요” 중심으로 잡는 것이 좋음

---

## 5. 경비 `/expenses`

```text
┌────────────────────────────────────────────────────────────────────┐
│ 경비청구                                                           │
│ 설명: 증빙 기반 청구 등록과 승인/지급 흐름 관리                  │
├────────────────────────────────────────────────────────────────────┤
│ KPI: 작성중 │ 승인대기 │ 지급대기 │ 반려 │ 증빙누락              │
├────────────────────────────────────────────────────────────────────┤
│ 상태 흐름                                                          │
│ DRAFT → SUBMITTED → IN_REVIEW → APPROVED → PAID                  │
├────────────────────────────────────────────────────────────────────┤
│ 주의 필요 카드                                                     │
│ - 영수증 누락                                                     │
│ - 프로젝트 미지정                                                  │
│ - 금액 이상치                                                      │
├────────────────────────────────────────────────────────────────────┤
│ 상단 폼 / 하단 최근 청구 목록                                     │
└────────────────────────────────────────────────────────────────────┘
```

### 의도
- 사용자는 “입력”보다 “어디서 막혔는지”를 알고 싶어함
- 상태 흐름을 시각적으로 드러내면 ERP 느낌이 강해짐

---

## 6. 직원 디렉터리 `/employees`

```text
┌────────────────────────────────────────────────────────────────────┐
│ 직원 디렉터리                                                      │
│ 설명: 직원 조회 + 운영 리스크 확인                                │
├────────────────────────────────────────────────────────────────────┤
│ KPI: 재직 │ 휴직 │ 비활성 │ 신규 입사 예정                        │
├────────────────────────────────────────────────────────────────────┤
│ 운영 알림                                                          │
│ - 서류 미완료                                                      │
│ - 권한 미설정                                                      │
│ - 부서 미지정                                                      │
├────────────────────────────────────────────────────────────────────┤
│ 필터/검색                                                          │
├────────────────────────────────────────────────────────────────────┤
│ 테이블 or 카드 보기                                               │
└────────────────────────────────────────────────────────────────────┘
```

### 의도
- 단순 인사 마스터가 아니라 HR 운영 허브로 확장할 기반 마련

---

## 7. 공통 패턴 정리

모든 핵심 화면에서 아래 블록을 재사용한다.

1. 헤더: 화면 목적 + 주요 액션
2. KPI 행
3. 경고/예외 블록
4. 필터/탭
5. 리스트/카드 본문
6. 세부 액션

---

## 8. 실제 구현 우선순위

1. `/workspace`
2. `/approvals`
3. `/attendance/ledger`
4. `/expenses`
5. `/employees`

이번 작업에서는 1, 2를 먼저 실제 반영 대상으로 잡는다.
