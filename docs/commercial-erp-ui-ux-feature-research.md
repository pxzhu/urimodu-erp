# 상용 ERP UI/UX · 기능 리서치 메모

작성일: 2026-03-18
대상 프로젝트: 우리모두 ERP
목적: 현재 ERP 프로젝트의 디자인/UI/UX 및 기능 구성이 만족스럽지 않다는 문제의식에 따라, 주요 상용 ERP의 제품 구조와 화면 설계 패턴을 참고해 개선 우선순위를 잡기 위한 리서치 문서

---

## 1. 이 문서의 결론부터

현재 우리모두 ERP가 즉시 참고해야 할 방향은 아래 5가지로 정리된다.

1. **모듈 나열형 UI에서 역할 중심 홈(Role-based Home)으로 전환**
2. **리스트 중심 화면에서 업무 흐름 중심 화면으로 전환**
3. **ERP 핵심 화면마다 KPI + 예외/알림 + 다음 액션을 한 화면에 같이 배치**
4. **데이터 입력 화면보다 승인/처리/예외해결 화면의 UX 완성도를 우선 강화**
5. **“무료 ERP”가 아니라 “한국 실무용 AI-native ERP”라는 제품 정체성이 보이도록 홈/네비게이션/자동화 UX를 재설계**

즉, 단순히 예쁜 UI가 아니라, 사용자가 “오늘 해야 할 일”이 바로 보이는 ERP 구조로 바꾸는 것이 핵심이다.

---

## 2. 현재 프로젝트 관점에서 보이는 문제

README 기준 현재 주요 화면은 다음과 같다.

- AI-Native ERP 랜딩
- 직원 디렉터리
- 문서 및 템플릿
- 결재함
- 근태 원장
- 경비청구

이 구성을 보면 모듈 자체는 나쁘지 않다. 다만 상용 ERP 관점에서 보면 아래 약점이 생기기 쉽다.

### 2.1 홈이 “브랜드 소개”에 가깝고 “업무 시작점” 느낌이 약해질 가능성
상용 ERP는 홈이 홍보 랜딩이라기보다 **개인 업무 허브**에 가깝다.
예를 들면:

- 오늘 승인할 결재 수
- 처리 지연 중인 청구
- 미제출 근태 정정
- 이번 주 입사 예정자/서명 대기 문서
- 재고/매출/미수금 같은 핵심 예외 지표

이런 요소가 홈 첫 화면에서 보여야 한다.

### 2.2 리스트는 있어도 “무엇을 먼저 해야 하는지”가 약할 수 있음
ERP에서 중요한 것은 데이터 보관보다 **업무 우선순위 제시**다.
예:

- 직원 디렉터리 → 단순 목록보다 “입사 예정/서류 미완료/권한 미설정”이 보여야 함
- 결재함 → 단순 상신 목록보다 “오늘 승인 필요/지연/반려 사유 대응 필요”가 강조돼야 함
- 근태 원장 → 단순 ledger보다 “이상 근태/누락/정정 대기”가 먼저 보여야 함

### 2.3 모듈 전환은 되지만 업무 플로우 연결감이 약할 가능성
상용 ERP는 모듈이 따로따로 있어도 사용자는 한 업무 흐름으로 느껴야 한다.
예:

- 직원 등록 → 서명/계약 → 근태 정책 할당 → 권한 부여
- 구매 요청 → 승인 → 발주 → 입고 → 회계 반영
- 문서 생성 → 승인 → 전자서명 → 보관 → 검색

지금 구조는 기능 모듈은 있으나, 화면 설계가 이 흐름을 얼마나 자연스럽게 보여주는지가 중요하다.

## 3. 상용 ERP별 리서치 요약

이번 리서치는 공식 제품 소개/기능 페이지 기준으로 정리했다.

대상:

- SAP Cloud ERP / S/4HANA 계열
- Microsoft Dynamics 365 Business Central
- Oracle NetSuite ERP
- Odoo
- Acumatica Cloud ERP

## 4. SAP Cloud ERP에서 참고할 점

공식 페이지에서 반복적으로 강조하는 포인트는 다음과 같았다.

- role-based, intuitive interface
- preconfigured best practices
- finance / procurement / supply chain / manufacturing / sales / service / asset 관리의 통합
- AI + automation + analytics의 기본 내장
- 확장 가능한 API 중심 구조

### 4.1 SAP식 UI/UX 핵심
SAP 계열은 기능이 많지만, 사용자에게는 보통 아래 구조로 보인다.

- 역할별 시작점
- 업무 카드/타일 기반 진입
- KPI와 예외 상황 먼저 노출
- drill-down 구조로 세부 화면 진입
- 전사 표준 프로세스 반영

즉, 사용자는 “메뉴를 탐색”하기보다 “내 역할의 업무 보드”에서 시작한다.

### 4.2 우리모두 ERP에 적용할 인사이트

#### 적용 포인트 1: 홈을 역할 기반 허브로 바꿔야 함
예:

- 관리자 홈
- 인사 담당자 홈
- 팀장 홈
- 일반 직원 홈

각 홈에서 보여줄 카드가 달라야 한다.

#### 적용 포인트 2: 데이터보다 예외를 먼저 보여줘야 함
예:

- 미승인 결재 14건
- 이번 주 휴가 충돌 3건
- 근태 누락 6건
- 미서명 문서 5건
- 처리 실패한 import/export 2건

#### 적용 포인트 3: “베스트 프랙티스 프로세스”를 제품 안에 녹여야 함
메뉴만 있는 것이 아니라, 업무 단계가 화면에서 자연스럽게 이어져야 한다.

## 5. Microsoft Dynamics 365 Business Central에서 참고할 점

공식 페이지 핵심 포인트:

- ERP는 finance, procurement, manufacturing, supply chain 같은 core process 자동화
- cloud + resilience + security
- AI agents가 payables, sales order monitoring, issue flagging 등을 지원
- Microsoft 365, Teams, Outlook, Excel, Copilot, Power Platform과의 강한 연결
- 중소중견기업이 빠르게 확장 가능한 운영 허브

### 5.1 Business Central식 UX 핵심
Business Central의 강점은 **Role Center** 개념이다.
즉, 사용자가 로그인하면 공통 대시보드가 아니라, 역할별로 다른 “작업 시작 화면”을 받는다.

화면 설계 특징:

- 상단: 역할별 핵심 KPI
- 중앙: 현재 해야 할 업무 큐
- 하단/측면: 관련 문서, 최근 활동, 연결된 기능
- 문서/목록/상세로 매우 빠르게 이동
- Microsoft 제품군과 자연스럽게 연결되는 작업 경험

### 5.2 우리모두 ERP에 적용할 인사이트

#### 적용 포인트 1: Role Center 패턴 도입
최우선 추천 항목이다.

예:

- **HR Role Center**: 입사/퇴사 예정, 근태 예외, 휴가 승인 대기, 미완료 인사 문서
- **Manager Role Center**: 승인 대기, 팀 근태 요약, 경비 승인, 팀 일정 충돌
- **Employee Role Center**: 내 할 일, 결재 요청 상태, 급여/증빙/휴가/문서 바로가기
- **Finance Role Center**: 미처리 전표, 경비 승인 대기, 지급 예정, 계정 예외

#### 적용 포인트 2: 외부 협업도 UX 일부로 봐야 함
ERP 내부 화면만 보는 게 아니라:

- Outlook/메일 연계
- Teams/알림 연계
- 캘린더/문서 링크 연계
- AI 요약/질의 응답

까지 연결되면 훨씬 “업무 플랫폼”처럼 느껴진다.

#### 적용 포인트 3: 단순 CRUD보다 “다음 행동 추천”이 중요
예:

- 근태 누락 발견 → 바로 정정 요청
- 결재 지연 감지 → 담당자에게 리마인드
- 경비 처리 지연 → 지급 예정 큐로 이동

## 6. Oracle NetSuite에서 참고할 점

공식 페이지 핵심 포인트:

- all-in-one, AI-powered cloud management suite
- accounting, orders, inventory, projects, production, supply chain, warehouse의 통합
- multiple subsidiaries / legal entities 지원
- real-time visibility와 tighter control
- order management, procurement, warehouse, supply chain 강점

### 6.1 NetSuite식 UX 핵심
NetSuite는 “전사 통합 가시성”을 굉장히 강하게 밀고 있다.
UX 측면에서는 다음이 핵심이다.

- 하나의 데이터 소스에서 여러 모듈이 움직임
- 실시간 지표/리포트/트랜잭션 drill-down
- 다법인/다통화/다조직 지원을 고려한 탐색 구조
- 재고/주문/구매/회계가 분리되지 않고 하나의 흐름처럼 이어짐

### 6.2 우리모두 ERP에 적용할 인사이트

#### 적용 포인트 1: 문서/승인/회계를 각자 모듈이 아니라 흐름으로 보여줘야 함
예:

- 경비청구 작성
- 승인 완료
- 회계 처리 대기
- 지급 완료

이 상태 흐름이 시각적으로 드러나야 한다.

#### 적용 포인트 2: 상태 배지와 lifecycle UI를 강화해야 함
ERP에서 중요한 것은 “이 데이터가 현재 어느 상태인가”다.

추천 표현:

- DRAFT
- PENDING
- IN_REVIEW
- APPROVED
- REJECTED
- POSTED
- PAID
- FAILED

이 상태를 표/상세/카드/타임라인 전부에서 일관되게 보여주는 것이 중요하다.

#### 적용 포인트 3: 요약 → 상세 drill-down을 촘촘하게 설계해야 함
예:

- 이번 달 승인 지연 12건
- 클릭
- 지연 건 목록
- 클릭
- 문서 상세/담당자/병목 단계 확인

이 흐름이 빠를수록 ERP가 “일 잘 되는 툴”처럼 느껴진다.

## 7. Odoo에서 참고할 점

공식 페이지 핵심 포인트:

- Kanban 기반 파이프라인
- drag and drop
- 일정/후속 활동 자동 계획
- 이메일/라이브챗/SMS/VoIP 같은 커뮤니케이션과 통합
- quote 작성/전환 UX가 매우 빠름
- custom dashboards / real-time metrics / automation 강조
- 앱 단위 확장 구조

### 7.1 Odoo식 UX 핵심
Odoo의 장점은 “가볍고 직관적인 작업 UX”다.
특히 중소기업 사용자가 쉽게 적응한다.

특징:

- 카드를 중심으로 한 시각적 상태 전환
- drag and drop
- 최소 입력으로 많은 작업 처리
- 모듈 간 연결이 빠름
- 앱 마켓 형태의 확장성

### 7.2 우리모두 ERP에 적용할 인사이트

#### 적용 포인트 1: 승인/문서/채용/태스크에 Kanban을 적극 도입
현재 ERP에서 Kanban 효과가 큰 영역은 다음과 같다.

- 결재함
- 채용/온보딩
- 문서 작성/검토/승인
- 이슈/요청 처리
- 고객/파트너 관리(향후)

#### 적용 포인트 2: “두세 번 클릭으로 끝나는 UX”를 늘려야 함
예:

- 경비청구 생성
- 휴가 신청
- 문서 상신
- 서명 요청 발송
- 근태 정정 요청

이런 작업은 복잡한 폼보다 빠른 행동 UX가 중요하다.

#### 적용 포인트 3: 커뮤니케이션 히스토리를 한곳에서 보이게 해야 함
ERP에서 사용자는 “누가 언제 뭐라고 했는지”를 자주 찾는다.
따라서:

- 승인 코멘트
- 반려 사유
- 문서 수정 요청
- 알림 로그
- 담당자 메모

가 각 화면에서 자연스럽게 보여야 한다.

## 8. Acumatica에서 참고할 점

공식 페이지 핵심 포인트:

- modern, adaptable cloud ERP
- SMB~mid-market 대상
- industry edition 개념이 강함
- financials, project accounting, CRM, reporting, BI 통합
- dashboards, self-service reports, visual scheduling, field/office 연결 강조

### 8.1 Acumatica식 UX 핵심
Acumatica는 “산업별 업무 흐름에 맞춘 ERP”를 강조한다.
그래서 범용 ERP지만 실제 화면은 vertical use case에 가깝다.

특징:

- industry-specific navigation
- 보고서/대시보드/분석을 사용자 스스로 구성 가능
- 현장과 사무실을 연결하는 실시간성 강조
- 프로젝트/원가/인력/재고/현장 작업이 하나의 컨텍스트에서 이어짐

### 8.2 우리모두 ERP에 적용할 인사이트

#### 적용 포인트 1: 업종/업무군별 템플릿 UX가 필요함
우리모두 ERP도 장기적으로는 아래처럼 나뉠 수 있다.

- 일반 사무형 회사
- 제조/유통형 회사
- 프로젝트형 회사
- 현장/서비스형 회사

초기부터 전부 만들 필요는 없지만, 정보 구조는 업종별 확장이 가능해야 한다.

#### 적용 포인트 2: 셀프 서비스 리포트/필터가 강해야 함
중소기업 ERP는 관리자보다 실무자가 더 많이 쓴다. 따라서:

- 저장 가능한 필터
- 사용자별 컬럼 설정
- 자주 보는 보기(view) 저장
- 나만의 대시보드 카드 고정

이런 기능이 중요하다.

#### 적용 포인트 3: 프로젝트/예산/실행/정산 흐름을 통합적으로 보여줘야 함
특히 경비/문서/근태는 프로젝트 단위와 연결되면 훨씬 ERP답게 느껴진다.

## 9. 상용 ERP에서 공통적으로 보이는 UI/UX 패턴

상용 ERP들을 비교해보면 공통 패턴은 상당히 명확하다.

### 9.1 공통 패턴 A: 역할 기반 시작 화면
모든 사용자가 같은 홈을 보지 않는다.
역할에 따라 KPI, 할 일, 최근 항목, 경고가 달라진다.

### 9.2 공통 패턴 B: KPI + Alert + Task가 한 화면에 같이 있음
예쁜 그래프만 있는 대시보드는 거의 없다.
실제로는:

- 숫자
- 이상 징후
- 바로 처리 버튼

이 함께 있어야 한다.

### 9.3 공통 패턴 C: 상태가 시각적으로 강하게 드러남
ERP는 상태 기반 소프트웨어다.
따라서 배지, 타임라인, 단계 표시, 큐 구분이 매우 중요하다.

### 9.4 공통 패턴 D: 리스트보다 흐름을 보여줌
목록은 기본이고, 그 위에 “현재 막힌 단계 / 다음 단계 / 책임자”가 보여야 한다.

### 9.5 공통 패턴 E: Drill-down이 빠름
요약 카드 → 목록 → 상세 → 액션까지 이동이 빨라야 한다.

### 9.6 공통 패턴 F: 확장성과 통합이 UX에 녹아 있음
ERP는 고립된 앱이 아니라 다른 업무 도구와 연결된다.
메일, 문서, 알림, 승인, AI, API가 하나의 경험처럼 보여야 한다.

---

## 10. 우리모두 ERP에 대한 구체적 개선 제안

여기부터는 실제 프로젝트에 바로 반영 가능한 제안이다.

## 10.1 정보 구조(IA) 재설계

현재 추정 구조:

- 직원
- 문서
- 결재
- 근태
- 경비

이 구조를 아래처럼 바꾸는 것이 더 낫다.

### 제안 정보 구조

#### 홈
- 내 할 일
- 승인 대기
- 오늘 일정/이슈
- 주요 지표
- 최근 활동

#### 사람
- 직원
- 조직도
- 입퇴사
- 권한/역할
- 근태/휴가

#### 문서/승인
- 문서함
- 템플릿
- 결재 상신
- 승인 대기
- 반려/보완 요청
- 전자서명

#### 재무
- 경비청구
- 지급 대기
- 계정/전표
- 예산/집행

#### 운영
- 가져오기/내보내기
- 감사 로그
- 연동
- 시스템 작업 현황

이렇게 묶으면 사용자가 모듈보다 업무 도메인 중심으로 이해하기 쉬워진다.

---

## 10.2 홈 화면 재설계

### 관리자 홈 제안
- 오늘 처리 필요 건수
- 지연 승인 건수
- 근태 예외 건수
- 실패한 배치/연동 건수
- 최근 감사 로그 이상 징후
- 빠른 실행: 직원 추가 / 문서 생성 / 결재 설정 / 가져오기 실행

### HR 홈 제안
- 입사 예정자
- 서류 미제출
- 근태 누락
- 휴가 승인 대기
- 정책 미할당 직원

### 일반 직원 홈 제안
- 내 결재 상태
- 오늘 출퇴근/근태 상태
- 사용 가능한 휴가
- 최근 받은 문서/서명 요청
- 빠른 실행: 휴가 신청 / 경비청구 / 문서 제출

핵심은 “예쁜 랜딩”보다 “바로 일하는 첫 화면”이다.

---

## 10.3 결재/문서 UX 재설계

결재는 ERP 만족도를 크게 좌우한다.

### 추천 패턴
- 상단: 승인 대기 / 긴급 / 반려 / 완료 탭
- 좌측: 상태 필터 + 문서 유형 필터
- 중앙: 카드 또는 리스트
- 우측 상세 패널: 문서 미리보기, 승인선, 코멘트, 첨부
- 액션: 승인 / 반려 / 보완 요청 / 위임

### 추가 제안
- 승인 지연 SLA 표시
- 반려 사유 템플릿
- 모바일 승인 최적화
- “다음 승인자” 표시
- 승인 경로 타임라인

이건 Odoo의 직관성과 NetSuite의 상태 관리, Business Central의 작업 큐 개념을 섞어서 만들면 좋다.

---

## 10.4 직원/조직 UX 재설계

직원 디렉터리는 단순 목록보다 **업무 처리 화면**이어야 한다.

### 추천 요소
- 신규 입사 예정 카드
- 필수 서류 미완료 인원
- 권한 미설정 인원
- 수습 종료 예정
- 부서별 인원/결원 현황

### 목록 화면에서 필요한 UX
- 저장 가능한 필터
- 컬럼 커스터마이즈
- 대량 액션
- 상세 패널 즉시 열기
- 조직도/카드/테이블 보기 전환

---

## 10.5 근태 UX 재설계

근태는 ledger만 보여주면 실무자가 피곤해진다.

### 추천 구조
- 오늘 출근 현황
- 이상 근태 요약
- 정정 요청 대기
- 휴가/출장/재택 현황
- 직원별 타임라인

### 추천 기능
- 지각/누락 자동 하이라이트
- 정정 요청 원클릭 진입
- 관리자 승인 큐
- 달력 보기 + 리스트 보기 동시 지원
- 정책 위반 경고

근태는 표보다 타임라인과 캘린더가 중요하다.

---

## 10.6 경비/재무 UX 재설계

경비청구는 “폼 작성”보다 “정산 상태 흐름”이 중요하다.

### 추천 패턴
- 작성 중
- 제출됨
- 승인 대기
- 회계 반영 대기
- 지급 완료
- 반려

이 단계를 각 카드와 상세 화면에서 일관되게 보여줘야 한다.

### 추천 기능
- 영수증 OCR 업로드(장기)
- 프로젝트/부서/계정 연결
- 승인 지연 경고
- 지급 예정일 표시
- 법인카드/개인결제 구분
- 증빙 누락 경고

---

## 10.7 디자인 시스템 측면 개선 포인트

상용 ERP 느낌이 안 나는 이유는 종종 기능보다 디자인 시스템 문제다.

### 꼭 필요한 시각 규칙
- 상태 배지 컬러 체계 통일
- 카드/리스트/패널의 계층 통일
- 밀도 높은 데이터 화면에서도 spacing 기준 유지
- 표 헤더/고정컬럼/필터바 패턴 통일
- destructive / primary / secondary 액션 구분 명확화
- light mode 기준 가독성 최우선

### 추천 톤
우리모두 ERP는 “한국 실무용”이라는 인상을 줘야 하므로,
지나치게 글로벌 SaaS식 감성보다 아래 톤이 어울린다.

- 신뢰감 있는 중립 계열
- 너무 장식적이지 않은 카드 UI
- 데이터 가독성 우선
- 업무 처리 버튼은 명확하고 단호하게
- 과한 그림보다 실무 지표 중심

---

## 11. MVP 우선순위 제안

전부 한 번에 바꾸면 무너진다. 우선순위는 아래가 좋다.

### P1. 바로 바꿔야 할 것
1. 역할 기반 홈 화면
2. 결재/문서 작업 큐 UX
3. 상태 배지/Lifecycle UI 통일
4. 근태 예외 중심 대시보드
5. 저장 가능한 필터/보기 설정

### P2. 다음 단계
1. Kanban 뷰 도입
2. 우측 상세 패널 패턴 도입
3. 빠른 액션/단축 흐름
4. 프로젝트/부서/비용 연결 강화
5. 알림/리마인드 UX

### P3. 장기 차별화
1. AI 업무 요약
2. AI 승인 보조/이상 감지
3. 커뮤니케이션 히스토리 통합
4. 업종별 Role Center 템플릿
5. 모바일 승인/현장 UX 최적화

---

## 12. 우리모두 ERP가 가져가야 할 포지셔닝

상용 ERP를 그대로 복제하면 안 된다.
우리모두 ERP는 아래 포지션이 가장 설득력 있다.

### 제안 포지셔닝
**한국 실무형 AI-native self-hosted ERP**

이 포지셔닝에 맞게 화면도 다음 질문에 답해야 한다.

- 오늘 내가 처리해야 할 일은 무엇인가?
- 어디가 막혀 있는가?
- 누구에게 넘겨야 하는가?
- 무엇이 자동화되는가?
- 한국 실무 문서/결재/근태 맥락을 얼마나 잘 이해하는가?

즉, “무료 ERP”보다 “실무에 맞는 ERP”가 더 강한 메시지다.

---

## 13. 바로 실행 가능한 다음 액션

이 문서를 바탕으로 추천하는 실제 실행 순서는 아래다.

1. 현재 웹 화면을 기준으로 **홈 / 결재 / 직원 / 근태 / 경비** UX 문제를 화면별로 분해
2. 각 화면에 대해 **KPI / Alert / Task / List / Detail / Action** 6블록 구조로 재설계
3. 관리자 / HR / 직원 기준 **Role Center 와이어프레임** 먼저 제작
4. 상태 배지, 필터바, 상세 패널, 액션바를 공통 컴포넌트로 디자인 시스템화
5. 이후에야 시각 polish 진행

---

## 14. 요약

상용 ERP들을 비교해보면, 좋은 ERP의 핵심은 아래로 요약된다.

- 홈은 소개 화면이 아니라 업무 허브여야 한다.
- 리스트보다 우선순위와 예외를 먼저 보여줘야 한다.
- 상태와 흐름이 시각적으로 명확해야 한다.
- 역할별 시작점이 달라야 한다.
- CRUD보다 다음 행동 추천이 중요하다.
- 확장성, 자동화, 협업 흐름이 UX에 녹아 있어야 한다.

우리모두 ERP는 이미 기능 모듈의 방향성은 좋다.
이제 필요한 것은 “모듈 나열”에서 “업무 흐름 중심 UX”로 넘어가는 일이다.

그 전환만 잘하면, 지금보다 훨씬 상용 ERP에 가까운 완성도를 만들 수 있다.
