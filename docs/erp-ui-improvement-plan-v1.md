# ERP UI 개편안 v1

작성일: 2026-03-18
기준 문서:
- `docs/commercial-erp-ui-ux-feature-research.md`
- `README.md`

목적:
상용 ERP 리서치를 바탕으로 우리모두 ERP의 웹 UI/UX를 실제 작업 가능한 단위로 재구성한다. 이 문서는 참고 메모가 아니라 **실행 계획서**다.

---

## 1. 문제 정의

현재 우리모두 ERP는 기능 모듈의 방향은 좋지만, 사용자가 처음 진입했을 때 아래 문제가 발생할 수 있다.

1. 홈이 업무 시작점보다 소개/진입 페이지에 가깝다.
2. 문서/결재/근태/경비 화면이 **업무 흐름**보다 **리스트/CRUD** 중심으로 느껴질 수 있다.
3. “오늘 처리할 일”, “막혀 있는 일”, “지연된 일”, “내가 다음에 해야 할 일”이 한눈에 들어오지 않는다.
4. ERP의 핵심인 상태(lifecycle)와 예외(exception)가 시각적으로 강하게 드러나지 않는다.
5. 한국 실무형 ERP라는 정체성보다 “기능 모듈 모음”처럼 보일 가능성이 있다.

---

## 2. 개편 목표

이번 UI 개편 1차 목표는 아래 다섯 가지다.

1. **Role Center 도입**
2. **KPI + Alert + Task + Queue 중심 홈 구성**
3. **결재/근태/경비의 상태 흐름 UI 강화**
4. **리스트 화면에 우선순위 및 예외 강조 추가**
5. **공통 액션/상태/필터 패턴 통일**

이번 단계에서는 “완성형 디자인 리뉴얼”이 아니라, 실제 사용성을 바꾸는 구조 개편을 우선한다.

---

## 3. 제품 UX 원칙

### 3.1 업무 허브 우선
첫 화면은 홍보가 아니라 일하는 화면이어야 한다.

### 3.2 예외 우선
정상 데이터보다 처리 지연, 누락, 검토 필요 항목을 먼저 보여준다.

### 3.3 상태 우선
ERP의 모든 핵심 엔티티는 상태가 중요하므로, 상태 배지/단계/타임라인을 통일한다.

### 3.4 다음 행동 우선
조회보다 액션이 빨라야 한다. 가능한 한 “다음 행동” 버튼을 같이 둔다.

### 3.5 한국 실무 우선
결재, 근태, 문서, 증빙, 승인선, 정정, 지급 대기 등 한국 실무에서 자주 보는 표현과 흐름이 자연스럽게 보여야 한다.

## 4. 화면별 개편 우선순위

## P1 (즉시 개편)

### A. 업무 홈 (`/workspace`)
가장 먼저 바꿔야 하는 화면이다.

#### 목표
- 관리자/직원별 시작점 차별화
- KPI + Alert + Quick Action + Task Queue 동시 제공
- 핵심 모듈 진입보다 “오늘 할 일”이 먼저 보이게 구성

#### 추가할 요소
- 역할 기반 헤더
- 운영 경고 카드
- 오늘 처리할 일 큐
- 승인/근태/경비/문서 상태 요약
- 빠른 실행 버튼
- 최근 활동/병목 현황

### B. 결재함 (`/approvals`)
ERP 만족도를 크게 좌우하는 핵심 화면.

#### 목표
- 단순 테이블에서 “우선순위 결재 큐”로 전환
- 지연/긴급/일반 항목 구분
- 단계/결재자/상태가 즉시 보이게 개선

#### 추가할 요소
- 결재 KPI
- 긴급/검토/일반 큐 분리
- 상태 배지 강화
- 코멘트/문서 흐름/다음 단계 표시

---

## P2 (다음 단계)

### C. 근태 원장 (`/attendance/ledger`)
#### 목표
- ledger 테이블 중심에서 예외 중심 화면으로 전환
- 검토 필요 항목 우선 노출

#### 추가할 요소
- 이상 근태 요약 카드
- 검토 필요 필터 기본값
- 캘린더/타임라인 확장 여지

### D. 경비 (`/expenses`)
#### 목표
- 신청/승인/지급 흐름이 시각적으로 드러나도록 개선
- 첨부 누락, 증빙 누락, 승인 대기 등 예외 표시 강화

#### 추가할 요소
- 상태별 카운트
- 첨부 누락 경고
- 빠른 제출 흐름 정리

### E. 직원 디렉터리 (`/employees`)
#### 목표
- 단순 목록에서 운영 중심 디렉터리로 확장
- 신규/비활성/권한 미설정/서류 미완료 같은 리스크 관점 도입

---

## P3 (중기)

- Kanban 뷰 도입 (결재/문서/요청)
- 우측 상세 패널 공통 패턴 도입
- 사용자 정의 대시보드 카드 저장
- 모바일 승인 UX 강화
- AI 요약/이상 감지 UI 도입

## 6. 공통 컴포넌트 개편안

이번 단계에서 공통화해야 할 디자인/UX 패턴:

1. `StatusBadge`
2. `MetricCard`
3. `AlertCard`
4. `TaskQueueCard`
5. `SectionHeader` (제목 + 설명 + 액션)
6. `PriorityPill` (긴급/검토/일반)
7. `EmptyState`
8. `FilterBar`

이 공통 패턴이 자리잡아야 화면마다 품질이 들쭉날쭉하지 않다.

---

## 7. 상태 체계 통일안

아래 상태는 표/카드/상세/타임라인에서 동일한 색/문구 규칙을 써야 한다.

- `DRAFT`
- `PENDING`
- `IN_REVIEW`
- `WAITING`
- `APPROVED`
- `REJECTED`
- `COMPLETED`
- `FAILED`
- `NEEDS_REVIEW`

### 컬러 방향
- Draft: 중립 회색
- Pending / Waiting: 파랑
- In Review: 보라 또는 진한 파랑
- Approved / Completed: 초록
- Rejected / Failed: 빨강
- Needs Review: 주황

## 8. 이번 작업 범위

이번 실행에서는 아래까지를 범위로 잡는다.

### 포함
- 리서치 문서 정리
- 와이어프레임 문서 작성
- `/workspace` Role Center 강화
- `/approvals` 우선순위 큐/상태 강화
- 최소한의 공통 스타일 추가

### 제외
- 전체 디자인 시스템 전면 교체
- 모든 페이지 동시 리뉴얼
- 모바일 전면 최적화
- 백엔드 데이터 모델 변경
- AI 자동화 기능 실제 구현

## 9. 개발 작업 순서

1. 리서치 문서 확정
2. 와이어프레임/기획 문서 작성
3. 홈(Role Center) 개편
4. 결재함 개편
5. 상태/카드 스타일 보강
6. QA 및 스크린샷 확인
7. PR 생성 및 머지

---

## 10. QA 기준

개편 후 아래를 확인한다.

1. 관리자 모드에서 홈 KPI/알림/빠른 작업이 정상 노출되는가
2. 일반 사용자 모드에서 홈 문구/작업 큐가 역할에 맞게 달라지는가
3. 결재함에서 긴급/검토/일반 구분이 자연스러운가
4. 상태 배지가 기존 번역/표현과 충돌하지 않는가
5. 데스크톱 화면에서 레이아웃 깨짐/overflow가 없는가
6. dark mode에서도 최소한의 가독성이 유지되는가

---

## 11. 성공 기준

이번 개편이 성공했다고 판단하는 기준은 다음과 같다.

- 첫 화면을 봤을 때 ERP가 “업무 허브”처럼 느껴진다.
- 결재함이 단순 목록이 아니라 “처리해야 할 큐”처럼 보인다.
- 사용자가 다음 행동을 찾는 시간이 줄어든다.
- 디자인이 더 상용 ERP다운 구조로 보인다.
- 이후 근태/경비/직원 화면으로 같은 패턴을 확장하기 쉬워진다.
